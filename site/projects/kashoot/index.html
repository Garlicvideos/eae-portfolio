Kashoot! is a complete rewrite of Kahoot! without the downsides. It retains all of Kahoot!'s current features, as well as prevent bots from joining game lobbies. There is also no max player cap, unlike the original game.
This project is currently a work-in-progress.
View this project on github
Technologies used
Node.JS
Node.JS was used to handle the backend. It processes many events, such as captcha verification, client connections, lobby events, and many more.
MongoDB
MongoDB was used to store information on game lobbies and player info. Such information includes the game pin, player ID, questions, answers, and a whole lot more.
Express.JS
Express.JS was used to assist in managing client connections to the webserver.
Socket.io
Socket.io was used to manage socket connections to the server. Information such as lobby details, connection status and authentication, are sent through Socket.io.
Highlights
Flood Prevention
Using Google's Recaptcha v2 Invisible, bots will be filtered out. Game lobbies will no longer be flooded by fake players, overloading servers and crashing them.
No hard max player capacity
The 1,000 player limit is removed. You can now host games as large as your computer can handle. School-wide kahoot events are now possible.
Easy to deploy
All you need to do is to start the server and it will do all the setting up for you. A game can be up and running within minutes!
(This feature is in the making)
Open Source
Anyone can easily and readily deploy a copy on their system. Programmers can also take a look at the source code and help improve or learn from it.
